MCP Packet Tracer is a [Model Context Protocol](https://modelcontextprotocol.io) server that bridges any LLM — GitHub Copilot, Claude, Codex, Cursor — to Cisco Packet Tracer. Describe a network in plain English and the server plans, validates, generates IOS configs, and deploys the topology directly into a running Packet Tracer instance in real time.

## Get Started in 3 Steps

<Steps>
  <Step title="Install the server">
    Clone the repo and install it as an editable Python package:
    ```bash
    git clone https://github.com/Mats2208/MCP-Packet-Tracer
    cd MCP-Packet-Tracer
    pip install -e .
    ```
  </Step>
  <Step title="Connect your MCP client">
    Register the server with your preferred AI client. For Claude Code:
    ```bash
    claude mcp add --scope user --transport stdio packet-tracer -- python -m packet_tracer_mcp --stdio
    ```
    See [Client Setup](/setup/client-setup) for VS Code, Cursor, Claude Desktop, and Codex CLI.
  </Step>
  <Step title="Ask your LLM to build a network">
    ```
    "Create a network with 3 routers, OSPF routing, DHCP on each LAN, and 2 PCs per router"
    ```
    The server plans → validates → generates → deploys the topology automatically.
  </Step>
</Steps>

<Note>
  Live deploy into a running Packet Tracer instance requires a one-time bridge setup. See [Live Deploy Setup](/setup/live-deploy) for the bootstrap snippet. The MCP tools work without the bridge too — you can plan, generate scripts, and export configs to disk.
</Note>
