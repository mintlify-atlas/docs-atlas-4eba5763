`pt_generate_script` turns a validated `TopologyPlan` (the JSON output of `pt_plan_topology`) into a ready-to-run JavaScript file for Cisco Packet Tracer's PTBuilder Script Engine. The script places every device on the logical canvas, installs expansion modules, draws cables between interfaces, and ‚Äî when `include_configs` is `true` ‚Äî embeds the full IOS CLI configuration as commented blocks so you can review everything in one file before running it.

## Parameters

<ParamField body="plan_json" type="string" required>
  The `TopologyPlan` JSON produced by `pt_plan_topology` or `pt_full_build`. Must contain a non-empty `devices` array.
</ParamField>

<ParamField body="include_configs" type="boolean" default="true">
  When `true`, the returned script appends all IOS CLI config blocks as `/* ... */` comments after the `addDevice` / `addLink` calls. Set to `false` to get a pure structural script with no configuration comments.
</ParamField>

## Return value

<ResponseField name="script" type="string">
  A JavaScript string ready to paste into the PT Builder Code Editor (Extensions ‚Üí Builder Code Editor) or send directly to a running Packet Tracer instance via `pt_live_deploy`. Each PTBuilder function call is on its own line.
</ResponseField>

## Script structure

The generator follows a strict three-phase ordering that PTBuilder's Script Engine requires: devices must exist before modules can be installed, and modules must be installed before any link can reference the ports they add.

```javascript
// Phase 1 ‚Äî place devices on the logical canvas
lwAddDevice("R1", 1, "2911", 100, 100);
lwAddDevice("SW1", 2, "2960-24TT", 100, 300);
lwAddDevice("PC1", 8, "PC-PT", 300, 300);

// Phase 2 ‚Äî install expansion modules (only present when the plan includes modules)
addModule("R1", "0/0", "HWIC-2T");

// Phase 3 ‚Äî draw cables
lwAddLink("R1", "GigabitEthernet0/0", "SW1", "GigabitEthernet0/1", "straight");
lwAddLink("SW1", "FastEthernet0/1", "PC1", "FastEthernet0", "straight");

// Phase 4 ‚Äî IOS configs as comments (include_configs=true)
/* --- R1 ---
enable
configure terminal
hostname R1
...
*/
```

<Note>
  The script uses `lwAddDevice` and `lwAddLink` (logical-workspace helpers injected by the runtime patches) rather than the raw `addDevice` / `addLink` globals. The helpers write directly to the logical canvas so devices and cables appear immediately without requiring a save-and-reload cycle in Packet Tracer.
</Note>

## Example script excerpt

The following excerpt shows what a two-device topology looks like after generation. The `configureIosDevice` and `configurePcIp` calls are generated by `generate_executable_script` (used internally by `pt_live_deploy`); when you call `pt_generate_script` with `include_configs=true`, the CLI content appears as comments instead.

```javascript
lwAddDevice("R1", 1, "2911", 100, 100);
addModule("R1", "0/0", "HWIC-2T");
lwAddDevice("SW1", 2, "2960-24TT", 100, 300);
lwAddLink("R1", "GigabitEthernet0/0", "SW1", "GigabitEthernet0/1", "straight");
configureIosDevice("R1", "enable\nconfigure terminal\nhostname R1\n...");
```

## Execution order requirement

PTBuilder's Script Engine is sequential and stateful. Calling `addLink` before both endpoint devices exist raises a runtime error. The generator enforces the required order automatically:

1. **`lwAddDevice`** ‚Äî creates the device on the logical canvas
2. **`addModule`** ‚Äî installs expansion modules (e.g. `HWIC-2T` for serial ports); must run after the device is placed and before any link references module-added ports
3. **`lwAddLink`** ‚Äî connects two interface ports with the specified cable type

## Relationship to `pt_live_deploy`

`pt_generate_script` only produces the script string ‚Äî it does not send anything to Packet Tracer. When you call `pt_live_deploy`, the server internally calls `generate_executable_script` (which combines structural calls with live `configureIosDevice` / `configurePcIp` calls instead of comments) and streams each line to the HTTP bridge on port 54321, where PTBuilder picks it up every 500 ms and executes it in the Script Engine.

Use `pt_generate_script` when you want to inspect or manually paste the script. Use `pt_live_deploy` when you want fully automated, real-time deployment into a running Packet Tracer instance.

<Tip>
  Pipe the output of `pt_generate_script` into `pt_deploy` to copy it to the Windows clipboard with step-by-step paste instructions ‚Äî no bridge required.
</Tip>
